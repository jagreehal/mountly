<section class="section">
  <div class="section-head">
    <span class="eyebrow">01 · The problem</span>
    <h2 class="section-title">
      Modern web apps ship too much JavaScript upfront.
    </h2>
  </div>
  <p class="section-lead" style="max-width: 70ch;">
    Component libraries load everything at once. Microfrontends are operationally heavy.
    Framework lazy-loading lacks standardised interaction patterns. There's no unified system for
    <em style="color: var(--text-strong); font-style: normal;"> "load rich UI only when the user actually needs it"</em>.
  </p>

  <div class="before-after sl-markdown-no-margin">
    <div class="before">
      <h3>Before mountly</h3>
      <ul>
        <li>Payment widget</li>
        <li>Video player</li>
        <li>Image lightbox</li>
        <li>Analytics panel</li>
        <li>Chat widget</li>
      </ul>
      <p class="result">All shipped on first paint. Slow TTI, heavy bundle.</p>
    </div>
    <div class="after">
      <h3>After mountly</h3>
      <ul>
        <li>Page shell (light)</li>
        <li>User hovers → load module</li>
        <li>→ fetch data in parallel</li>
        <li>→ mount into shadow DOM</li>
      </ul>
      <p class="result">Fast TTI. Each widget pays only on intent.</p>
    </div>
  </div>
</section>

  <div class="section-head">
    <span class="eyebrow">02 · Triggers</span>

      Six ways to say <em style="font-style: italic; color: var(--accent);">"now"</em>.

  <p class="section-lead">
    A trigger is the signal that moves a feature from <code>idle</code> to <code>preload</code> to
    <code>mount</code>. Pick the one that matches user intent — or compose your own with a plugin.
  </p>

  <div class="trigger-grid sl-markdown-no-margin">
    <div class="trigger-cell">
      <span class="index">01</span>
      <span class="label">on intent</span>
      <span class="name">hover</span>
      <span class="desc">Preload on mouseenter, mount on click. The default for discoverable controls.</span>
    </div>
    <div class="trigger-cell">
      <span class="index">02</span>
      <span class="label">explicit</span>
      <span class="name">click</span>
      <span class="desc">Mount on click without preloading. Use when intent is committed, not exploratory.</span>
    </div>
    <div class="trigger-cell">
      <span class="index">03</span>
      <span class="label">accessibility</span>
      <span class="name">focus</span>

    <div class="trigger-cell">

<section class="section">
  <div class="section-head">
    <span class="eyebrow">03 · Lifecycle</span>
    <h2 class="section-title">
      One state machine, five states. The same shape on every framework.
    </h2>
  </div>
  <p class="section-lead">
    Every feature moves through the same sequence regardless of trigger or framework. You can hook into any phase, abort in flight, and unmount cleanly.
  </p>

  <div class="lifecycle sl-markdown-no-margin">
    <div class="lifecycle-step">
      <span class="step-num">01</span>
      <span class="step-name">idle</span>
      <span class="step-desc">Nothing loaded. Waiting for the trigger.</span>
    </div>
    <div class="lifecycle-step">
      <span class="step-num">02</span>
      <span class="step-name">preload</span>
      <span class="step-desc">Module fetched, data fetch may begin in parallel.</span>
    </div>
    <div class="lifecycle-step">
      <span class="step-num">03</span>
      <span class="step-name">activate</span>
      <span class="step-desc">Commitment signal. Finalise data, prepare to mount.</span>
    </div>
    <div class="lifecycle-step">
      <span class="step-num">04</span>
      <span class="step-name">mount</span>
      <span class="step-desc">Widget rendered into the container's shadow root.</span>

    <div class="lifecycle-step">

{/* ─── Code ─── */}

<section class="section">
  <div class="section-head">
    <span class="eyebrow">04 · The whole API</span>
    <h2 class="section-title">
      A widget. A feature. A trigger. Three lines you'll write.
    </h2>
  </div>
  <p class="section-lead">
    Components stay components. The adapter wraps them as widgets. <code>createOnDemandFeature</code> adds the on-demand lifecycle. <code>attach</code> wires it to a DOM element.
  </p>

```ts title="signup-card.ts"
import { createWidget } from 'mountly-react';
import SignupCard from './SignupCard.tsx';
import styles from './SignupCard.css?inline';

// 1. Wrap a component as a framework-agnostic widget
export default createWidget(SignupCard, { styles });
```

```ts title="page.ts"
import { createOnDemandFeature } from 'mountly';

// 2. Add the on-demand lifecycle around a widget
const signup = createOnDemandFeature({
  moduleId: 'signup-card',
  loadModule: () => import('./signup-card.js'),
  render: ({ mod, container, props }) => mod.mount(container, props),
});

// 3. Attach it to a DOM trigger — preload on hover, mount on click
signup.attach({
  trigger: document.querySelector('#cta')!,
  preloadOn: 'hover',
  activateOn: 'click',
});
```

> **Styling** — CSS is auto-loaded with the module, injected into the widget's shadow root, no FOUC.
